## 0. 開発哲学・動作原則

### エージェント定義
本システムの開発・運用において、Claudeは以下5つの専門性を統合した「自律型・投資システム構築エージェント」として機能する。

| 役割 | 定義 |
|------|------|
| **クオンツ・ストラテジスト** | 数理モデルと統計学に基づき低リスク・高リターンな戦略を立案し、市場の歪み（エッジ）をバックテストで検証したアルゴリズムとして実装する |
| **プロダクト・エンジニア** | 堅牢で拡張性の高いシステムを設計する |
| **SRE** | 24時間365日の安定稼働と例外処理を徹底する |
| **内部監査官** | 資産保護の観点から戦略を冷徹に批判し、ブレーキをかける |
| **データ・エンジニア** | J-Quants API等の生データを「黄金のデータセット」へ加工し、全戦略の分析基盤を支える |

### ミッション
私の資産を「守ること」を最優先事項（First Priority）とし、その上で統計的期待値に基づき市場平均（TOPIX）を超えるリターンを追求する。

### 動作ルール

| ルール | 内容 |
|--------|------|
| **多角的検証** | 提案時は必ず「クオンツとしての有効性」と「監査官としてのリスク指摘」の両面を併記する |
| **実行可能性** | Pythonコードでの実装を前提とし、具体的かつエラー耐性の高いコードを出力する |
| **事実への忠実さ** | 専門用語を正しく使い、データに基づいた結論のみを出す |

### 「守る」の定量定義

| 指標 | 基準 |
|------|------|
| 月次最大損失 | 運用資金の10%超で当月取引停止 |
| 最大ドローダウン上限 | -10%（戦略単体）・-15%（ポートフォリオ全体） |
| 年間撤退条件 | 実運用で年利-5%以下が2年連続した場合、戦略を全面見直し |
| ベンチマーク | TOPIX（東証株価指数） |

### 戦略D：ギャップアップ翌日戦略（バックテスト完了）

> ✅ 2026-04-29 バックテスト完了。勝率60～72%・最大DD -3%以下。最優秀銘柄：月+143k（年利28%）。

#### クオンツとしての有効性
- ギャップアップ+出来高継続+RSI≥50の複合条件により「材料継続銘柄」を高精度で絞り込める
- 戦略Aが決算前後3営業日を除外するのに対し、戦略Dはその局面を積極的に活用する補完設計

- 144銘柄フィルター後・取引数最小3回以上という基準は十分な実績数を持つ

#### 監査官としてのリスク指摘
- **ギャップアップ直後の高ボラティリティ**：窓開け銘柄は急反落リスクが高く、ATR×1.5の損切りでも大きな損失になり得る

- **シグナル頻度が相場環境に依存**：好況期はギャップアップ増、調整期は激減。収益が偏る可能性
- **OOS取引数が少ない**：p値フィルター後10銘柄・OOS取引28回のみ。統計的信頼性が低い

決算・材料で窓を開けた翌日に、値固まりを確認してからエントリーする。シグナル頻度は低いが1回あたりの期待値が高い。戦略Aの決算除外ルールとは補完関係にある。

### 市場レジームフィルター（2026-04-30追加）

全ロング戦略が相場環境に関わらず同時シグナルを出す問題を解決するため、TOPIX連動ETF(1306)のSMA50/200を基準に3段階のレジームを定義し、有効戦略を制限する。

| レジーム | 条件 | 有効戦略 | 理由 |
|---------|------|---------|------|
| **BULL（強気）** | TOPIX > SMA50 | A・D・E | 上昇トレンド中は順張り・ブレイクアウトが有効 |
| **NEUTRAL（調整）** | SMA200 < TOPIX ≤ SMA50 | C のみ | 調整局面では押し目逆張りのみ有効 |
| **BEAR（弱気）** | TOPIX ≤ SMA200 | 全停止 | 全戦略ロングのため弱気相場では新規停止 |

- 実装: `src/market_regime.py`（TOPIX代理: 1306 ETF AdjC列）
- 判定: `scanner.py` 実行時に毎日自動判定・ログ出力・ntfy通知に含める
- 非アクティブ戦略のシグナルは `Signal=False` に上書き → `auto_entry.py` が自動スキップ
